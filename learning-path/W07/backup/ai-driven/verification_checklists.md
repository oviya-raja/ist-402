# Verification Checklists - Template
# Use these checklists before marking tasks/phases complete

## TASK VERIFICATION CHECKLIST

Before marking any task complete, verify ALL of the following:

### Completion Verification
- [ ] Task actually completed (not just started)
- [ ] All subtasks/requirements met
- [ ] Deliverable matches task description
- [ ] No placeholder text or TODOs remain

### Quality Verification
- [ ] Meets minimum quality standards (check `learnings/quality_improvements.md`)
- [ ] Professional formatting and presentation
- [ ] No obvious errors or typos
- [ ] Follows best practices from global learnings
- [ ] Quantitative data included (metrics, statistics, percentages) when possible
- [ ] Visual aids considered (diagrams, maps, charts) if needed
- [ ] Specific context provided (not just general descriptions)
- [ ] Comparative analysis included (if relevant to task)

### Requirement Verification
- [ ] Aligned with requirements from `Assignment_Analysis_Prompt.md`
- [ ] Meets rubric criteria (if applicable)
- [ ] Addresses all specified criteria
- [ ] No missing components

### Documentation Verification
- [ ] Properly documented (if required)
- [ ] Files saved in correct locations
- [ ] Naming conventions followed
- [ ] Screenshots captured (if required)

### Self-Assessment
- [ ] Self-assessment completed
- [ ] Quality score assigned (1-5)
- [ ] Potential issues identified
- [ ] Improvement suggestions noted

### Approval Status
- [ ] Human approval obtained
- [ ] Feedback incorporated (if any)
- [ ] Ready to proceed to next task

## PHASE VERIFICATION CHECKLIST

Before marking any phase complete, verify ALL of the following:

### Task Completion
- [ ] All tasks in phase are completed
- [ ] All tasks have human approval
- [ ] No skipped or deferred tasks
- [ ] Task completion count matches tracker

### Phase Objectives
- [ ] All phase objectives met (from `Assignment_TODO_Tracker.md`)
- [ ] Phase deliverables complete
- [ ] Quality standards met for all tasks
- [ ] No critical issues remaining

### Quality Verification

- [ ] Professional presentation
- [ ] Follows assignment requirements
- [ ] Meets rubric standards

### Documentation
- [ ] All required documentation complete
- [ ] Screenshots captured (if required)

- [ ] Progress tracked accurately

### Rubric Alignment
- [ ] Phase work aligns with rubric
- [ ] All criteria addressed
- [ ] No gaps identified
- [ ] Ready for evaluation

### Phase Evaluation
- [ ] Phase evaluation form generated
- [ ] Self-assessment completed
- [ ] Quality score assigned

- [ ] Human approval obtained

### Learning Capture

- [ ] Patterns identified
- [ ] Feedback captured
- [ ] Improvements noted

## RUBRIC ALIGNMENT CHECKLIST

Before requesting approval, verify alignment with rubric:

### Read Rubric
- [ ] Read relevant rubric section from `Assignment_Analysis_Prompt.md`
- [ ] Understand all criteria
- [ ] Identify point values
- [ ] Note required evidence

### Check Each Criterion
- [ ] Criterion 1: [Description] - ✅/❌
- [ ] Criterion 2: [Description] - ✅/❌
- [ ] Criterion 3: [Description] - ✅/❌
- [ ] (Continue for all criteria)

### Evidence Verification
- [ ] Required evidence present
- [ ] Screenshots included (if needed)
- [ ] Documentation complete
- [ ] Examples provided (if needed)

### Gap Identification
- [ ] Any missing criteria identified
- [ ] Gaps documented
- [ ] Plan to address gaps
- [ ] Timeline for completion
