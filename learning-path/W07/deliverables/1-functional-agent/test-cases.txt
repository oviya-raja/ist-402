TEST CASE 2: Use Case 2 - Check Job Qualification
--------------------------------------------------

INPUT:
I found this job posting and want to know if I'm qualified:

Job Title: Software Engineer I - Backend
Company: Tech Company

Required Qualifications:
- Bachelor's degree in Computer Science or related field
- 1+ years of software development experience
- Proficiency in Python
- Experience with REST API development
- Experience with PostgreSQL or similar database
- Strong problem-solving skills

Preferred Qualifications:
- Experience with AWS
- Experience with Docker
- Experience with microservices architecture

Location: Remote or San Francisco, CA

My Profile:
- Education: BS Computer Science (graduating May 2025)
- Experience: 1.5 years (internships + projects)
- Skills: Python (Advanced), Java (Intermediate), AWS (Intermediate), SQL (Advanced), Git (Advanced)
- Location: Prefer remote or San Francisco

EXPECTED OUTPUT CHECKLIST:
- [ ] Agent calculates fitment percentage (should be 75-85%)
- [ ] Agent identifies matched skills (Python, AWS, SQL, etc.)
- [ ] Agent identifies missing/gaps (Docker, microservices)
- [ ] Agent provides recommendation (Apply/Consider/Not Recommended)
- [ ] Agent explains why (fitment breakdown)
- [ ] Agent uses fitment calculation methodology from knowledge base
- [ ] Agent provides actionable next steps
- [ ] Tone is supportive and encouraging

VERIFICATION CRITERIA
=====================

For each test case, verify:

1. FUNCTIONALITY:
   - [ ] Agent performs the requested task correctly
   - [ ] Agent uses knowledge base appropriately
   - [ ] Agent follows system prompt instructions

2. ACCURACY:
   - [ ] Fitment calculations are reasonable
   - [ ] Information from knowledge base is accurate
   - [ ] Recommendations make sense

3. PERSONALITY & TONE:
   - [ ] Supportive and encouraging
   - [ ] Clear and student-friendly
   - [ ] Action-oriented
   - [ ] Empathetic

4. MENTAL HEALTH FOCUS:
   - [ ] Includes mental health benefit messaging
   - [ ] Reduces overwhelm
   - [ ] Provides clarity
   - [ ] Focuses on actionable steps

5. ERROR HANDLING:
   - [ ] Handles errors gracefully
   - [ ] Provides helpful guidance
   - [ ] Asks for clarification when needed
   - [ ] Never just says "error"
